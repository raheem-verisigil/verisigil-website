Runtime Guard — Verify Before Execution

VeriSigil Runtime Guard is the operational trust layer that sits between an AI agent's decision and its execution. Every action is verified before it runs — identity checked, trust scored, policy applied, decision logged.

⚠️ The Problem Runtime Guard Solves

Autonomous AI agents make decisions and execute actions — payments, data access, API calls, deletions — without any verification of whether they should. There is no layer that asks: is this agent who it claims to be? Is this action within policy? Should a human approve this first? Runtime Guard is that layer.

⚡ How Runtime Guard Works

1
Agent initiates an action
Before executing anything — payment, database write, API call, tool use — the agent calls Runtime Guard first.
POST /v1/guard/verify
2
Identity verified cryptographically
VeriSigil checks the agent's Ed25519 signed passport. Confirms the agent is real, active, not expired, not cloned by another system (Shadow Detection™).
3
Trust score evaluated in real time
Each agent has a live trust score (0.0–1.0) updated by verifications, security scans, and threat events. Score below 0.60 = blocked automatically.
4
Policy applied to the action
Action-specific rules enforced automatically — payments above $1,000 escalated, dangerous tools blocked, PII access requires GDPR certification.
5
Decision returned in under 50ms
One of three decisions returned with confidence score, reason, and execution ID. Every decision cryptographically logged to immutable audit trail.
✅
ALLOW
Identity verified, trust sufficient, action within policy. Proceed.
👤
REQUIRE_HUMAN_APPROVAL
Risky action or policy threshold exceeded. Human must approve first.
🚫
DENY
Agent revoked, expired, cloned, or trust too low. Block immediately.
Integration — 3 lines of code
import requests, os

def verify_before_execution(action_type, details, resource):
    r = requests.post(
        "https://verisigil-api-production.up.railway.app/v1/guard/verify",
        headers={"x-api-key": os.getenv("VERISIGIL_API_KEY")},
        json={"agent_id": "vsa_your_agent", "action_type": action_type,
              "action_details": details, "resource": resource}
    ).json()
    if r["decision"] == "DENY":
        raise PermissionError(r["reason"])
    return r["decision"]  # ALLOW or REQUIRE_HUMAN_APPROVAL

# Example — before a $5,000 payment
verify_before_execution("payment", {"amount_usd": 5000}, "stripe_api")
# Returns: REQUIRE_HUMAN_APPROVAL — Payment $5000 exceeds auto-allow threshold
