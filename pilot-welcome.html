SECTION 02
Integration Checklist — 4 Steps
1
Add Runtime Guard call before your action executes
One API call. Add it before any sensitive action in your agent code.
POST /v1/guard/verify Headers: x-api-key: your_api_key Body: { agent_id, action_type, action_details, resource }
import requests, os

def verify_before_execution(action_type, details, resource):
    r = requests.post(
        "https://verisigil-api-production.up.railway.app/v1/guard/verify",
        headers={"x-api-key": os.getenv("VERISIGIL_API_KEY")},
        json={"agent_id": "your_agent_id", "action_type": action_type,
              "action_details": details, "resource": resource}
    ).json()
    if r["decision"] == "DENY":
        raise PermissionError(r["reason"])
    return r
2
Handle approval_url in REQUIRE_HUMAN_APPROVAL response
When Runtime Guard returns REQUIRE_HUMAN_APPROVAL — pause execution and surface the approval URL to your team. The approver will receive an email automatically.
result = verify_before_execution("payment", {"amount_usd": 5000}, "stripe")

if result["decision"] == "ALLOW":
    execute_payment()  # proceed normally

elif result["decision"] == "REQUIRE_HUMAN_APPROVAL":
    # Email sent automatically to approver
    # approval_url is in the response
    pause_execution(result["approval_url"])

elif result["decision"] == "DENY":
    block_and_log(result["reason"])  # blocked
3
Test with flow-test.html before going live
Run the full end-to-end test in your browser. Click Run, approve the action, verify the audit trail. Confirms everything works before connecting your real agent.
verisigilai.com/flow-test.html
4
Go live — connect your real agent
Replace the test agent ID with your real agent ID. Point at production. Every real action now goes through Runtime Guard before executing.
Replace: "your_agent_id" → your real vsa_xxxxxxxx Set env: VERISIGIL_API_KEY=your_key Done.
